#### 1. 在交互模式提示符下，定义一个列表，`L=[0,1,2,3]`。实验一些边界情况：
##### a. 索引值超过边界时，会发生何事（例如，`L[4]`）？
```shell
>>> L[4]
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
    L[4]
    ~^^^
IndexError: list index out of range
```
会触发索引错误`IndexError`，提示列表索引超出范围。

##### b. 分片运算超出边界时又如何（例如，`L[-1000:100]`）？
```shell
>>> L[-1000:100]
[0, 1, 2, 3]
```
会返回列表中的所有元素，因为分片操作会自动调整边界值，使其在有效范围内。

##### c. 最后，如果你试着反向抽取序列，也就是较低边界值大于较高边界值（例如 ，`L[3:1]`）Python会怎么处理？提示：试着赋值至此分片（`L[3:1]=['?']`）,  看看此值置于何处。你觉得这和分片超出边界是相同的现象吗？
```shell
>>> L[3:1]
[]
```
当较低边界值大于较高边界值时，Python会返回一个空列表。因为切片操作的默认步长是`1`，导致`3:1:1`没有任何符合条件的索引，返回的列表中的空列表。
```shell
>>> L[3:1] = ['?']; L
[0, 1, 2, '?', 3]
```
对切片赋值会替换切片的内容，但如果切片为空，则会在切片的起始位置插入新元素。
我认为和分片超出边界的情况不太一样。

##### d. 列表`L`赋值一个空列表给其偏移值之一（例如，`L[2]=[]`）。发生什么事？然后，赋值空列表给分片（`L[2:3]=[]`）。现在发生了什么事？
```shell
L[2] = []; L
[0, 1, [], '?', 3]
```
把空列表赋值给一个索引会将该索引位置的元素替换为一个空列表。
```shell
L[2:3] = []; L
[0, 1, '?', 3]
```
把空列表赋值给一个切片会触发[c.](#c-最后如果你试着反向抽取序列也就是较低边界值大于较高边界值例如-l31python会怎么处理提示试着赋值至此分片l31--看看此值置于何处你觉得这和分片超出边界是相同的现象吗)中的列表替换情况，把切片指定的元素替换为空的内容，即删除了切片指定的元素。

##### e. `del`叙述会删除偏移值、键、属性以及名称。将其用在列表上来删除一个元素（例如，`del L[0]`）。如果你删除整个分片，会发生何事（`del L[1:3]`）？当你赋值非序列给分片时，会发生什么变化（`L[1:2]=1`）？
```shell
>>> del L[0]; L
[1, '?', 3]
```
`del`某个索引会删除该索引位置的元素。
```shell
>>> del L[1:3]; L
[1]
```
`del`一个切片会删除切片指定的元素。
```shell
>>> L[1:2] = 1; L
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
    L[1:2] = 1; L
    ~^^^^^
TypeError: must assign iterable to extended slice
```
给切片赋值一个非序列会触发类型错误`TypeError`，提示必须将可迭代对象赋值给切片。
