#### 2. 列表解析（列表推导式）
**用三种做法编写代码来创建新列表，求列表`[2,4,9,16,25]`中所有数字的平方根：将它先写成for循环，然后是`map`调用，最后是列表解析。使用内置`math`模块中的`sqrt`函数来进行计算（导入`math`, 编写`math.sqrt(x)]`）。**
```python
import math


x = [2, 4, 9, 16, 25]


def forLoop(x: list) -> list:
    '''用for循环实现'''
    result = []
    for i in x:
        result.append(math.sqrt(i))
    return result


def mapFunction(x: list) -> list:
    '''用map函数实现'''
    return list(map(lambda i: math.sqrt(i), x))


def listComprehension(x: list) -> list:
    '''用列表推导式实现'''
    return [math.sqrt(i) for i in x]


func = [forLoop, mapFunction, listComprehension]
[print(f.__doc__, f(x)) for f in func]
```
输出如下：
```shell
用for循环实现 [1.4142135623730951, 2.0, 3.0, 4.0, 5.0]
用map函数实现 [1.4142135623730951, 2.0, 3.0, 4.0, 5.0]
用列表推导式实现 [1.4142135623730951, 2.0, 3.0, 4.0, 5.0]
```


#### 3. 编写基本循环
##### a. 写个for循环，打印字符串`S = "Spam"`中每个字符的ASCII码。使用内置函数`ord`（character）把每个字符转换成ASCII整数（在交互模式下测试来观察其工作方式）。
```shell
>>> S = "Spam"
>>> for c in S:
...     print("'" + c + "'的ASCII码是", ord(c))
... 
'S'的ASCII码是 83
'p'的ASCII码是 112
'a'的ASCII码是 97
'm'的ASCII码是 109
```
`ord`函数会返回一个字符的ASCII码整数值。
在交互模式下输入`for`行的`:`后，按回车键进入循环体时，交互环境会自动给出`...`，提示继续输入循环体的内容，直到给出一个空行结束循环体的输入。

##### b. 接着，修改循环来计算字符串中所有字符的ASCII码的总和。
```shell
>>> asciiSum = 0
>>> for c in S:
...     asciiSum += ord(c)
...
>>> asciiSum
401
```

##### c. 最后，再次修改代码，来返回一个新的列表，其中包含了字符串中每个字符的ASCII码。表达式`map(ord,S)`是否有类似的效果?
```shell
>>> asciis = []
>>> for c in S:
...     asciis.append(ord(c))
... 
>>> asciis
[83, 112, 97, 109]
>>> map(ord, S)
<map object at 0x000001E220FC9240>
```
两者的效果不完全相同，`map`函数会返回一个`map`对象而非一个列表。`map`对象是一个惰性的可迭代对象，避免一次性生成整个`map`转换结果列表，提高代码的运行性能。
可以用`list`函数把`map`对象转换成一个列表：
```shell
>>> list(map(ord, S))
[83, 112, 97, 109]
```

#### 5. 如下代码的输出是什么？为什么？
##### 5.a
```shell
>>> def func(a, b=4, c=5):
...     print(a, b, c)
... 
>>> func(1, 2)
```
输出是`1 2 5`。因为函数定义中为参数`b`和`c`指定了默认值，当调用函数时只提供了参数`a`和`b`的值，参数`c`使用了默认值`5`。所以`a`、`b`和`c`的值分别是`1`、`2`和`5`。

##### 5.b
```shell
>>> def func(a, b, c=5):
...     print(a, b, c)
... 
>>> func(1, c=3, b=2)
```
输出是`1 2 3`。因为在调用函数时，`a`的值被位置形式指定为`1`，而`b`和`c`的值被关键字形式分别指定为`2`和`3`。

##### 5.c
```shell
>>> def func(a, *pargs):
...     print(a, pargs)
... 
>>> func(1, 2, 3)
```
输出是`1 (2, 3)`。因为函数定义中使用`*`时，会以元组形式收集所有位置参数，除了第一个参数`a`以外的参数都被收集到`pargs`中，形成一个元组`(2, 3)`。而后`pargs`被以元组的形式打印了出来。

##### 5.d 
```shell
>>> def func(a, b, c=3, d=4):
...     print(a, b, c, d)
... 
>>> func(1, *(5, 6))
```
输出是`1 5 6 4`。因为在调用函数时，`a`的值被位置形式指定为`1`，`*(5, 6)`被解包为两个位置参数，指定了`b`和`c`的值，参数`d`使用了默认值`4`。


#### 6. 下面的代码会输出什么？为什么？
##### 6.a 
```shell
>>> X = 'Spam'
>>> def func():
...     print(X)
... 
>>> func()
```
输出是`Spam`。因为函数`func`在定义时就已经捕获了变量`X`的值，当调用函数时，函数会使用这个捕获的值来打印`X`。

##### 6.b
```shell
>>> X = 'Spam'
>>> def func():
...     X = 'NI!'
... 
>>> func()
>>> print(X)
```
输出是`Spam`。因为函数`func`内部定义的变量`X`只在函数的局部作用域内有效，并不会影响到函数外部的变量`X`。因此，当打印函数外部的变量`X`时，仍然打印`Spam`。

##### 6.c
```shell
>>> X = 'Spam'
>>> def func():
...     X = 'NI!'
...     print(X)
...
>>> func()
>>> print(X)
```
输出是：
```shell
NI!
Spam
```
和[6.b](#6b)一样，函数内部的变量`X`只在函数的局部作用域内有效，影响了函数内部的打印语句，输出了`NI!`；而在函数外部打印`X`时仍然打印`Spam`。

##### 6.d
```shell
>>> X = 'Spam'
>>> def func():
...     global X
...     X = 'NI!'
... 
>>> func()
>>> print(X)
```
输出是`NI!`。因为在函数`func`中使用了`global`关键字声明了变量`X`为全局变量，这使得函数修改的`X`就是全局的`X`，因此当打印的`X`时，输出了修改后的值`NI!`。

##### 6.e
```shell
>>> X = 'Spam'
>>> def func():
...     X = 'NI!'
...     def nested():
...         print(X)
...     nested()
... 
>>> func()
>>> X
```
输出是：
```shell
NI!
'Spam'
```
与[6.b](#6b)同理，函数`func`内部定义的变量`X`影响了其中的`nested`函数，使之打印了内部`X`，输出了`NI!`；而在函数外部打印变量`X`时，仍然打印了全局的`Spam`。注意此时是使用`>>> X`直接访问`X`，所以会打印出全局变量`X`的值`'Spam'`。

##### 6.f
```shell
>>> def func():
...     X = 'NI!'
...     def nested():
...         nonlocal X
...         X = 'Spam'
...     nested()
...     print(X)
... 
>>> func()
```
输出是`Spam`。因为函数`nested`中使用了`nonlocal`关键字声明了变量`X`，使得这个`X`指向了最邻近的`func`函数中的`X`，因此打印的`func`中的`X`时，输出了修改后的值`Spam`。


#### 7. 思考下面三条语句。它们会改变`A`打印出的值吗？
##### 7.a
```python
A = "spam"
B = A
B = "shrubbery"
```
不会。
1. 字符串是不可变对象，`A = "spam"`这个语句创建了一个字符串对象`"spam"`，并将变量`A`指向它。
2. `B = A`这个语句使得`B`也指向了同一个字符串对象。
3. 执行`B = "shrubbery"`这个语句时，创建了一个新的字符串对象，并将变量`B`指向这个新的对象，而`A`仍然指向原来的字符串对象`"spam"`。因此，`A`打印出的值仍然是`"spam"`。

##### 7.b
```python
A = ["spam"]
B = A
B[0] = "shrubbery"
```
会。
1. 列表是可变对象，`A = ["spam"]`这个语句创建了一个列表对象（其中有一个字符串对象作为元素），并将变量`A`指向它。
2. `B = A`这个语句使得`B`也指向了同一个列表对象。
3. 执行`B[0] = "shrubbery"`这个语句时，修改了列表对象中的第一个元素，将其从`"spam"`修改为`"shrubbery"`。由于`A`和`B`指向同一个列表对象，所以当打印`A`时，输出的值是`["shrubbery"]`。

##### 7.c
```python
A = ["spam"]
B = A[:]
B[0] = "shrubbery"
```
不会。
1. `A = ["spam"]`这个语句创建了一个列表对象（其中有一个字符串对象作为元素），并将变量`A`指向它。
2. `B = A[:]`这个语句访问了`A`中的所有元素，并用它们创建了一个新的列表对象。
3. 执行`B[0] = "shrubbery"`这个语句时，修改了新列表对象中的第一个元素，将其从`"spam"`修改为`"shrubbery"`。由于`A`和`B`指向不同的列表对象，所以当打印`A`时，输出的值仍然是`["spam"]`。
